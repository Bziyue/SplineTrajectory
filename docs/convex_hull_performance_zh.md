# Bezier / MINVO 凸包表示性能报告

## 测试环境

- CPU：AMD Ryzen 9 7945HX，16 核 32 线程
- 编译器：GCC 15.2.0
- 编译参数：`-O3 -march=native -ffast-math -DNDEBUG`
- Eigen，单线程
- 基准固定在 CPU 4：`taskset -c 4`
- 输入：`PPolyND<3>`，每段时长约为 0.75–1.25 s
- 每个结果是 9 个批次的中位数；每批次自动增加迭代次数，直到持续约 18 ms
- 正向时间包含表示对象、控制点及其元数据的分配和构造
- 反向时间包含输出系数梯度、时长梯度的分配和清零
- 不包含轨迹采样、避障代价计算或 MINCO 的后续 `propagateGrad()`

原始数据位于
[`docs/convex_hull_benchmark.csv`](convex_hull_benchmark.csv)，可执行基准位于
[`benchmarks/benchmark_convex_hull_basis.cpp`](../benchmarks/benchmark_convex_hull_basis.cpp)。

复现命令：

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --target benchmark_convex_hull_basis -j2
taskset -c 4 ./build/benchmark_convex_hull_basis \
  docs/convex_hull_benchmark.csv
```

可以减少这部分分配成本。

## 导数表示

`PPolyND::derivative()` 只生成新的幂系数，因此明显快于完整凸包转换。在
64 段五次三维轨迹上：

| 操作 | 总耗时 [µs] | 每段耗时 [ns] |
| :--- | ---: | ---: |
| `PPolyND::derivative(1)` | 0.421 | 6.58 |
| `PPolyND::derivative(2)` | 0.352 | 5.50 |
| Bezier 位置表示 | 6.307 | 98.5 |
| Bezier 速度表示 | 5.376 | 84.0 |
| Bezier 加速度表示 | 4.214 | 65.8 |
| MINVO 位置表示 | 6.215 | 97.1 |
| MINVO 速度表示 | 5.380 | 84.1 |
| MINVO 加速度表示 | 4.206 | 65.7 |

直接调用 `toBezier(ppoly, r)` 或 `toMINVO(ppoly, r)` 不会先构造一个中间
`PPolyND::derivative(r)`。导数因子、\(T^k\) 归一化和基变换在一次循环中
完成。导数次数越高，输出次数和控制点数量越少，因此速度、加速度表示比
位置表示更快。

## Bezier 细分

以下使用 64 段五次三维轨迹。深度 \(s\) 表示每个源段经过 \(s\) 层二分，
产生 \(2^s\) 个 Bezier piece。

| 深度 | piece 数 | 控制点数 | 正向 [µs] | 反向 [µs] |
| ---: | ---: | ---: | ---: | ---: |
| 0 | 64 | 384 | 6.201 | 5.826 |
| 1 | 128 | 768 | 11.164 | 11.450 |
| 2 | 256 | 1536 | 20.366 | 21.711 |
| 3 | 512 | 3072 | 39.462 | 41.894 |
| 4 | 1024 | 6144 | 85.574 | 88.466 |

从深度 1 开始，控制点数量每增加一倍，正向和反向时间也接近增加一倍。
深度 4 时正向约为 13.9 ns/输出控制点，反向约为 14.4 ns/控制点。反向
使用

\[
G_\mathrm{parent}=L^TG_\mathrm{left}+R^TG_\mathrm{right}
\]

逐层合并，不保存“每个叶子到源幂系数”的大矩阵，内存和时间都与生成的
控制点数量线性相关。

Bezier 与 MINVO 的运行时几乎一致。两者最终都是一个很小的稠密基矩阵与
归一化幂系数相乘；MINVO 的矩阵已在编译期数据中提供，运行时不求逆、不
读取 MAT 文件。

## 复杂度和结论

令源段数为 \(N\)，导数后的次数为 \(m\)，维数为 \(D\)，Bezier 细分深度
为 \(s\)。

- 不细分正向转换：\(O(Nm^2D)\)；
- 不细分反向传播：\(O(Nm^2D)\)；
- 二分正向和反向的附加开销：\(O(N2^s m^2D)\)；
- 输出控制点内存：\(O(N2^s mD)\)；
- 基矩阵仅保存一份，不再为每个源段分配一个矩阵。

对常用的五次三维轨迹，64 段位置转换和反传合计约 12 µs；二层细分后
合计约 42 µs。相对于一次完整轨迹优化中的 ESDF、走廊距离和优化器迭代，
该转换与伴随开销通常较小。
